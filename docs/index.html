PRR

GitHub Pull Request에 AI 리뷰어가 코멘트를 남기는 도구. Claude Code 스킬로 동작하며, 유료 API 없이 ToS를 완전히 준수한다.

비용 0원 Claude Code 스킬 ToS 완전 준수 Private repo 지원
// 핵심 제약

설계 원칙

◈
비용 0원
유료 API 없음. Claude Code 구독 범위 내에서만 동작. 외부 서비스 의존 없음.
◉
ToS 완전 준수
사람이 직접 트리거하는 Claude Code 스킬 방식. 자동화 파이프라인이 없어 Consumer ToS §3-7 위반 없음.
◎
기능 최소화
설정 관리는 CLI, AI 작업은 스킬. 두 인터페이스의 역할을 명확히 분리.

시작하기

사전 요구사항

terminal
$claude --version  # Claude Code CLI
$gh auth status   # GitHub CLI 인증 확인
$python3 --version # Python 3.10+

PATH 등록

~/.zshrc
$echo 'export PATH="$PATH:/path/to/PRR"' >> ~/.zshrc
$source ~/.zshrc
$prr status  # 환경 확인

스킬 등록

terminal
$cp -r ~/PRR/skills/prr-review ~/.claude/skills/
$cp -r ~/PRR/skills/prr-scan ~/.claude/skills/
$cp -r ~/PRR/skills/prr-add-reviewer ~/.claude/skills/

스킬 파일(~/.claude/skills/prr-*/SKILL.md) 안의 PRR_DIR 경로를 실제 클론 경로로 수정해야 한다. which prr | xargs dirname 으로 확인.

전체 워크플로우

1
리포 등록
CLI로 설정 디렉터리를 생성하고 기본 리뷰어를 준비한다.
prr init owner/my-repo
2
프로젝트 스캔
Claude Code에서 스킬을 실행해 env.json을 자동 생성한다.
/prr-scan ~/projects/my-repo owner/my-repo
3
리뷰어 설정
AI가 프로젝트에 맞는 리뷰어 후보를 추천한다. 선택하면 페르소나를 자동 구성해 저장한다.
/prr-add-reviewer owner/my-repo
4
PR 리뷰 실행
PR 번호를 지정해 스킬을 실행하면 GitHub에 코멘트가 게시된다.
/prr-review owner/my-repo 42
// CLI 인터페이스

커맨드 목록

prr CLI는 설정 관리에 집중한다. AI 작업(스캔, 리뷰)은 Claude Code 스킬이 담당한다.

prr --help
$prr init <owner/repo>           # 리포 등록
$prr reviewer add <owner/repo>     # 리뷰어 추가
$prr reviewer list <owner/repo>    # 리뷰어 목록
$prr repo list                  # 등록된 리포 목록
$prr status                      # 환경 상태 확인
// prr init

리포 등록

설정 디렉터리와 기본 리뷰어 파일을 생성한다. 이후 /prr-scan 스킬로 env.json을 생성하면 된다.

예시
$prr init owner/my-repo
prr init owner/repo ├─ configs/repos/owner__repo/ 생성 ├─ reviewers/junior.json → 기본 리뷰어 템플릿 복사 └─ 다음 단계 안내 출력 ├─ /prr-scan <local-path> owner/repo → env.json 생성 └─ prr reviewer add owner/repo → 리뷰어 추가
// prr reviewer

리뷰어 관리

한 리포에 여러 리뷰어를 설정할 수 있다. 각 리뷰어는 독립적인 페르소나로 별도의 GitHub 코멘트를 게시한다.

인터랙티브
$prr reviewer add owner/my-repo
>리뷰어 ID: senior
>에디터로 열까요? [Y/n]
목록 확인
$prr reviewer list owner/my-repo
✓ junior enabled
✓ senior enabled
// Claude Code 스킬

/prr-scan

로컬 프로젝트를 분석해 env.json을 자동 생성한다. Claude Code가 직접 파일을 읽고 프로젝트 특성을 파악해 JSON을 출력한다.

Claude Code 스킬
/prr-scan ~/projects/my-repo owner/my-repo

파일판별 정보

/prr-add-reviewer

프로젝트 env.json을 읽어 기술 스택에 맞는 리뷰어 후보를 추천한다. 선택하면 페르소나·집중 항목·무시 항목을 자동으로 구성해 저장한다.

Claude Code 스킬
/prr-add-reviewer owner/my-repo
/prr-add-reviewer owner/repo │ ├─ env.json 로드 → 언어·프레임워크·의존성 파악 ├─ 기존 리뷰어 목록 확인 │ ├─ 리뷰어 후보 4–6개 추천 출력 (프로젝트 특성 기반) │ 예) 1. senior — 아키텍처·패턴 전반 │ 2. netcode — 네트워크 동기화·RPC 전문가 │ 3. performance — GC·풀링 최적화 전문가 │ ├─ 사용자 선택 ├─ persona / focus / ignore 자동 구성 → JSON 출력 ├─ 사용자 확인 [Y/n] └─ reviewers/{id}.json 저장
// Claude Code 스킬

/prr-review

등록된 모든 활성 리뷰어가 순서대로 PR을 분석하고, 각자 별도의 GitHub 코멘트를 남긴다.

Claude Code 스킬
/prr-review owner/my-repo 42
/prr-review owner/repo 42 │ ├─ env.json 로드 → exclude_paths, on_update 파악 ├─ gh pr diff 42 --repo owner/repo │ └─ exclude_paths 패턴 매칭으로 파일 블록 제거 │ ├─ on_update="review" 시 │ └─ 이전 PRR 코멘트 삭제 (<!-- PRR-AUTO-REVIEW --> 마커 기반) │ ├─ [junior.json] 리뷰어 ──────────────────────── │ ├─ 페르소나 채택 (focus / ignore / severity) │ ├─ diff 검토 → 이슈 목록 작성 │ └─ gh pr comment → GitHub 코멘트 게시 │ └─ [senior.json] 리뷰어 ──────────────────────── ├─ 페르소나 채택 (focus / ignore / severity) ├─ diff 검토 → 이슈 목록 작성 └─ gh pr comment → GitHub 코멘트 게시
// 설정

env.json

리포 환경 정보와 리뷰 동작을 정의한다. /prr-scan 스킬이 자동 생성하며, 이후 수동으로 수정할 수 있다.

configs/repos/owner__repo/env.json
{ "repo": "owner/repo-name", "languages": ["TypeScript", "CSS"], "runtime": "Node.js 20", "framework": "Next.js 14", "test_framework": "Jest", "linter": "ESLint (airbnb-typescript)", "key_dependencies": ["prisma", "zod", "tailwindcss"], "conventions": [ "서버 컴포넌트 우선, use client 최소화", "함수형 컴포넌트만 사용" ], "review": { "exclude_paths": ["migrations/", "*.generated.ts", "*.lock"], "max_diff_lines": 500, "on_update": "skip" // "skip" | "review" } }
on_update동작
"skip"이전 PRR 코멘트를 유지 (기본값)
"review"이전 PRR 코멘트 삭제 후 새로 리뷰
// 설정

reviewer.json

리뷰어의 페르소나, 집중 항목, 무시 항목을 정의한다. /prr-review 스킬이 이 파일을 읽어 리뷰 방식과 코멘트 형식을 결정한다.

configs/repos/owner__repo/reviewers/junior.json
{ "id": "junior", "enabled": true, "comment_header": "🌱 Junior Reviewer", "persona": "당신은 1년차 주니어 개발자입니다...", "focus": [ "변수명·함수명이 역할을 명확히 드러내는가", "명백한 로직 오류나 누락된 예외처리", "불필요한 중복 코드" ], "ignore": [ "시스템 아키텍처 결정", "성능 최적화", "보안 취약점" ], "severity_threshold": "low", // low | medium | high "lgtm_comment": true, "tone": "친근하고 격려하는 말투. 동료 개발자를 응원하는 느낌으로.", "comment_sections": ["review_basis", "praise", "issues", "summary"] }

comment_sections 옵션

값섹션 내용
"review_basis"집중 항목(focus)과 검토 제외 항목(ignore) 요약
"praise"diff에서 잘 작성된 코드·패턴 언급. 칭찬할 것이 없으면 섹션 생략
"issues"발견된 이슈 목록. 이슈가 없으면 ✅ LGTM 메시지로 대체
"summary"tone에 맞는 마무리 한마디

tone과 comment_sections는 선택 필드. 없으면 ["issues"]와 중립 말투를 사용한다.

예외 처리

1
Diff가 클 때
exclude_paths로 lock 파일, 생성 코드, 빌드 산출물을 사전 제거한다. Claude Code는 대용량 컨텍스트를 네이티브로 처리하므로 별도 분할 없이 전체 diff를 검토한다.
exclude_paths Claude 네이티브 처리
2
PR 재리뷰
on_update: "skip"이면 이전 코멘트를 그대로 유지 (기본값). "review"로 설정하면 <!-- PRR-AUTO-REVIEW --> 마커로 이전 PRR 코멘트를 식별·삭제하고 새로 리뷰한다.
skip review
3
LGTM (이상 없음)
리뷰어가 문제를 찾지 못하면 투명하게 LGTM 코멘트를 게시한다. lgtm_comment: false로 설정 시 코멘트 없이 조용히 통과.
LGTM ✓
4
리뷰 실패 시
gh CLI 오류나 설정 누락 시 스킬 내에서 즉시 에러 메시지로 피드백한다. macOS 데스크탑 알림(notify.sh)으로 추가 인지 가능.
에러 메시지 macOS 알림
