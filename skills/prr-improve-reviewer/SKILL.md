## 변수
- `TARGET_ID` = 첫 번째 인수 (없으면 모든 리뷰어 대상)
- `CONFIG_DIR` = `$(pwd)/.prr`
- `$CONFIG_DIR/env.json` 없으면 오류 후 종료

## 점검 대상 로드
`env.json`에서 프로젝트 컨텍스트(언어·프레임워크·의존성·컨벤션) 파악.
`$CONFIG_DIR/reviewers/`에서 대상 JSON 파일 로드. 파일 없으면 오류 후 종료.

## 리뷰어별 품질 점검

각 리뷰어를 아래 기준으로 평가한다.

**persona**
- 역할·경험·관점이 구체적인가? "시니어 개발자입니다"처럼 막연하면 개선 필요.

**focus**
- 항목이 실제로 검토 방향을 좁혀주는가? "코드 품질" 같은 포괄적 항목은 구체화 필요.
- 프로젝트 기술 스택의 실제 리스크를 반영하는가?

**rules**
- 각 규칙이 diff에서 pass/fail 판단 가능한가? "확인한다" 형태는 위반 조건으로 재작성 필요.
- 이 리뷰어의 focus 범위에서 자주 발생하는 패턴 중 rules에 빠진 것이 있는가?
- env.json의 `conventions`·`key_dependencies`와 연관된 규칙이 반영됐는가?

**ignore**
- 이 리뷰어가 다루지 않아야 할 항목이 명확히 제외됐는가?

**severity_threshold / tone / comment_sections**
- 리뷰어 역할에 맞는 설정인가?

## 개선안 출력

리뷰어마다 아래 형식으로 출력한다:

```
─── {id} ──────────────────────────────
현재 상태: {간단한 평가 한 줄}

개선 제안:
  [persona] {현재값} → {제안값}
  [focus]   {추가/수정할 항목}
  [rules]   + {추가 제안 규칙 1}
             + {추가 제안 규칙 2}
             ~ {기존 규칙} → {더 명확한 표현}
  [ignore]  + {추가 제안}
```

변경 없이 양호한 항목은 `✓ {필드} 양호` 한 줄로만 표기한다.

## 확인 및 저장

리뷰어 전체 개선안 출력 후:
```
위 개선안을 적용할까요?
  a) 전체 적용
  b) 리뷰어별 선택
  n) 취소
```

`b` 선택 시 리뷰어마다 `[Y/n]` 확인.
각 항목별 선택은 하지 않는다 — 리뷰어 단위로만 적용.

적용 시 기존 파일을 개선안으로 교체해 `$CONFIG_DIR/reviewers/{id}.json`에 저장 (들여쓰기 2칸).

완료: `✓ {N}개 리뷰어 개선 완료` 출력.
