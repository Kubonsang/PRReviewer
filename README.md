# PRR — PR Auto Reviewer

An AI-powered code reviewer that posts comments on GitHub Pull Requests.
Runs as an AI coding tool skill with zero API cost, fully compliant with ToS.

**[한국어 README](README.ko.md)**

---

## Overview

- **AI Engine**: AI coding tool skills (`/prr-scan`, `/prr-add-reviewer`, `/prr-review`, `/prr-followup`)
- **Skill Management**: `prr` CLI (`skill sync`, `status`)
- **PR Integration**: `gh` CLI
- **Cost**: Free (within your AI coding tool subscription)
- **ToS**: Human-triggered only → fully compliant with Anthropic & GitHub ToS

Define reviewer personas in JSON and each posts separate GitHub comments.
Issues are posted as inline comments on specific diff lines; summaries go as a separate PR comment.
Multiple reviewers (e.g. junior, senior, security) can be configured per repo.

Config files (`.prr/`) are stored inside each project folder.
All skills are run from **within the project directory** in your AI coding tool.

## Prerequisites

- [Claude Code](https://claude.ai/code) installed and logged in
- [GitHub CLI](https://cli.github.com) installed and authenticated (`gh auth login`)
- Python 3.10+

## Installation

```bash
git clone https://github.com/Kubonsang/PRReviewer.git ~/PRR
echo 'export PATH="$PATH:$HOME/PRR"' >> ~/.zshrc
source ~/.zshrc
```

Register skills:

```bash
prr skill sync
```

Verify environment:

```bash
prr status
```

## Usage

Open your AI coding tool from the **project directory** you want to review.

### 1. Scan + Register Project

Run from the project root. Auto-generates `.prr/env.json`:

```
/prr-scan
```

### 2. Add Reviewers

AI recommends reviewers tailored to your project's tech stack:

```
/prr-add-reviewer
```

### 3. Review a PR

```
/prr-review 42
```

### 4. Follow Up (optional)

After a fix commit is pushed, posts reply comments on resolved issues:

```
/prr-followup 42
```

## Config Files

Config is stored in `.prr/` inside each project root.

### `.prr/env.json`

Repo environment info (auto-generated by `/prr-scan`).

```json
{
  "repo": "owner/repo",
  "languages": ["TypeScript"],
  "framework": "Next.js 14",
  "key_dependencies": ["prisma", "zod"],
  "conventions": ["server components first"],
  "review": {
    "exclude_paths": ["*.lock", "*.generated.ts"],
    "on_update": "skip"
  }
}
```

### `.prr/reviewers/{id}.json`

Reviewer persona.

```json
{
  "id": "junior",
  "enabled": true,
  "comment_header": "🌱 Junior Reviewer",
  "persona": "You are a junior developer with 1 year of experience.",
  "focus": ["logic errors", "duplicate code"],
  "ignore": ["architecture decisions", "performance optimization"],
  "severity_threshold": "low",
  "lgtm_comment": true,
  "tone": "Friendly and encouraging. Supportive like a teammate.",
  "comment_sections": ["review_basis", "praise", "issues", "summary"]
}
```

`comment_sections` values:
- `"review_basis"` — summary of focus and ignore items
- `"praise"` — highlights well-written code (omitted if nothing to praise)
- `"issues"` — found issues (replaced with LGTM if none)
- `"summary"` — closing remark matching `tone`

`tone` and `comment_sections` are optional. Defaults: `["issues"]` with neutral tone.

`on_update` values:
- `"skip"` — keep existing PRR comments (default)
- `"review"` — delete previous PRR comments and re-review
